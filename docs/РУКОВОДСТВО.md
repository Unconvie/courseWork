# Руководство по коду visionCore

# Краткая карта: что где лежит и что править

---

## 1. Общая схема (поток данных)

```
main.cpp
  │  парсит CLI: какой файл/папка, сохранять ли noiseFree / shapeOutline
  │
  ├─ FileWalker ──► список путей к .png/.jpg …
  │
  ├─ ImageIO::read ──► Image
  │
  └─ Pipeline::process(Image)
        1. INoiseFilter      → result.denoised   (для *-noiseFree)
        2. IDetector         → список Detection  (контуры, bbox)
        3. IFeatureExtractor → FeatureVector     (на каждый объект)
        4. IClassifier       → Classification    (Circle 40%, …)
        5. IAnnotator        → result.annotated  (для *-shapeOutline)
  │
  └─ ImageIO::write ──► файлы рядом с исходником
```

**Машинного обучения нет:** классификация — правила в `src/domain/shapes/*.cpp` (`match()`).

## 2. Структура папок

| Путь | Назначение |
|------|------------|
| `src/main.cpp` | Точка входа: CLI, цикл по файлам, сохранение, лог в консоль |

| `src/pipeline/impl/` | OpenCV: фильтр, детектор, признаки, классификатор, разметка |
| `include/visionCore/domain/` | Detection, FeatureVector, Classification, ShapeKind |
| `src/domain/shapes/` | Эталоны форм: `Circle::match`, `Triangle::match`, … |

| `.vscode/tasks.json` | Сборка и запуск из VS Code |

## 3. Какой файл обрабатывается

Задаётся **первым аргументом** при запуске:

```text
visionCore_app.exe <путь> [опции]
```

- **Файл** `data\samples\тест1.png` → обрабатывается только он.
- **Папка** `data\samples` → все подходящие изображения (см. `FileWalker`).

Код: `main.cpp` → `parseArgs()` → `walker.collect(opts.inputPath)`.

Скрипт `scripts/run_samples.bat` жёстко указывает `data\samples\тест1.png`.

**Не обрабатываются** (пропускаются автоматически):
- `*-noiseFree*`, `*-shapeOutline*`, `*-copy` — см. `PathUtils::isDerivedOutput`.

**Текстовый отчёт** `*-report.txt`:
- по умолчанию **вкл.** — `AppConfig.hpp` → `saveReportByDefault = true`;
- CLI: `--no-report` выключить, `--save-report` включить принудительно.

## 4. Куда сохраняются копии

Сейчас **всегда в ту же папку, что и исходник**, имя:

| Флаг CLI | Суффикс | Пример |
|----------|---------|--------|
| `--save-noise-free` | `-noiseFree` | `тест1-noiseFree.png` |
| `--save-outline` (по умолчанию) | `-shapeOutline` | `тест1-shapeOutline.png` |

Формула: `src/io/PathUtils.cpp` → `makeDerivedPath(source, "noiseFree")`.

Отдельной папки вывода (`--output-dir`) **пока нет** (план фаза 4). Чтобы сохранять в другое место, меняй `processFile()` в `main.cpp` или расширь `makeDerivedPath`.

## 5. Настройки чувствительности

Файл: **`include/visionCore/infra/AppConfig.hpp`**

После правки — пересборка (`scripts\build_msvc.bat` или задача CMake в VS Code).

| Группа полей | За что отвечает |
|--------------|----------------|
| `bilateralDiameter`, `gaussianSigma`, `medianKernel` | Как выглядит `*-noiseFree` |
| `useClahe`, `claheClipLimit` | Видимость бледных линий |
| `minContourArea`, `minCannyContourArea`, `minFaintContourArea` | Минимальный размер объекта (меньше → больше мусора) |
| `cannyLow/High`, `faintCannyLow/High` | Чувствительность к границам |
| `useHoughCircles`, `houghMinDist`, `houghParam2`, `houghMin/MaxRadius` | Отдельные круги, в т.ч. пересекающиеся |
| `nestedMaxAreaRatio` | Убрать «квадраты внутри круга» |
| `contourDedupeIou` | Слияние дубликатов bbox |
| `minClassificationConfidence` | Порог «Unknown» в классификации |
| `contourApproxEpsilon` | Сколько вершин у контура (влияет на Triangle/Hexagon) |
| `jpegQuality` | Качество сохранённых JPG |

Объект `AppConfig` создаётся в `main.cpp` и передаётся в `PipelineFactory::createDefault(config, …)`.

## 6. Классификация (не ML)

1. **`ContourFeatureExtractor`** — из контура: `vertexCount`, `circularity`, `aspectRatio`, `area`.
2. **`ShapeClassifier`** — для каждой формы вызывает `shape->match(features)`, нормализует в проценты.
3. **`src/domain/shapes/*.cpp`** — правила «насколько похоже на круг/треугольник/…».

Добавить новую форму:
- enum в `ShapeKind.hpp` + `toString` в `ShapeKind.cpp`;
- класс в `include/visionCore/domain/shapes/` + `src/domain/shapes/`;
- регистрация в `ShapeClassifier.cpp` (`m_shapes.push_back`).

Подписи на картинке: **`OpenCvAnnotator.cpp`** — цвета `colorFor()`, текст `"Circle 40%"` из `toString(kind)` и `topConfidence()`.

## 7. Графический интерфейс (GUI)

Один exe `visionCore_app.exe`:

| Запуск | Режим |
|--------|--------|
| Без аргументов или `--gui` | Окно с настройками и кнопкой «Запустить» |
| `visionCore_app.exe <путь> [флаги]` | Консоль (как раньше) |

В GUI: выбор файла/папки, все поля `AppConfig`, галочки noiseFree / shapeOutline / report,
после обработки — надпись **«Работа завершена»** и журнал.

Первая сборка с GUI скачивает GLFW и ImGui (нужен интернет при `cmake --preset msvc-debug`).

PowerShell «кракозябры» — перед запуском: `chcp 65001` или `scripts\run_samples.bat`.
